# CLAUDE.md — JSU/NCSY Wrapped Widget

This file is for AI agents working on this repo. Read it before making changes.

## What this project is

A static, embeddable Spotify-Wrapped-style story experience for JSU/NCSY chapters, regions, and national. No backend, no build step, no login. GitHub Pages hosts the static assets; WordPress embeds the widget on ncsy.org.

## Key architecture patterns

### UMD module (jsu-wrapped.js)
```js
(function (root, factory) {
  var api = factory(root);
  if (typeof module === "object" && module.exports) { module.exports = api; }
  if (root && root.document) { root.JSUWrapped = api; }
})(typeof globalThis !== "undefined" ? globalThis : this, function (root) { ... });
```
This runs in the browser as `window.JSUWrapped` and in Node for QA scripts. All internal functions are private; only the return value is exported. Written in ES5 for broad browser compatibility — use `var`, not `const`/`let`.

### Data flow
```
sample-wrapped-2026.json  →  findStoryRecord()  →  createEffectiveRecord()  →  createCards()  →  renderStory()
wrapped-config-2026.json  →  resolveStoryConfig()  ↗                        ↗
```
Config layers merge in this order: `defaults → national → region → program → chapter → variant`. Each layer only overrides what it sets.

### Config override system
`wrapped-config-2026.json` supports per-scope overrides:
- `hidden_cards`: array of card IDs to skip
- `card_overrides`: per-card copy fields (`headline`, `subtext`, `eyebrow`, etc.)
- `record_overrides`: correct a metric value (e.g., `chapter_persona: "The Spark"`)
- `custom_cards`: inject text/metric/media screens at placement positions
- `variants`: named alternate versions (e.g., `donor-recap`)
- `footprint_label`: national footprint card subtext override
- `brand_logo`, `palette`, `cta_label`, `cta_target`, `cta_href`

### CSS isolation
All selectors in `jsu-wrapped.css` must start with `#jsu-wrapped` or `#jsu-wrapped-wordpress-shell`. The `runCssIsolationSmoke` test in `qa-smoke.js` enforces this. The only exceptions are the `html:has(...)` and `body:has(...)` rules at the top of the file.

### wordpress-inline-embed.html
**Do not edit this file directly.** It is auto-generated by `sync-wordpress-inline.js` which splices the current `jsu-wrapped.css` and `jsu-wrapped.js` into the WordPress shell markup. After changing either JS or CSS file, always run:
```bash
node sync-wordpress-inline.js
```
The production check enforces this with `git diff --exit-code wordpress-inline-embed.html`.

### Cache token
A shared string (`jsuw-prod-20260619a`) is embedded in all hosted asset URLs. When JS, CSS, data, or config changes need to force a refresh:
```bash
node bump-cache-token.js jsuw-prod-YYYYMMDDx
```
This updates 15+ files atomically. Run `node check-production.js` afterward.

## How to run QA

```bash
node check-production.js        # full gate: sync, lint, validate, smoke, drift check
node validate-wrapped-data.js   # data/config contract only
node qa-smoke.js                # unit smokes only
node junior-wrapped-derive.test.js  # junior teen derivation unit tests
node hosted-smoke.js            # post-deploy: fetch live GitHub Pages URLs
node launch-audit.js            # launch-gate summary
```

GitHub Actions runs `check-production.js` on every push and pull request, with an enforced render smoke using the Ubuntu runner's system Chrome. All of these are equivalent checks — `check-production.js` runs the others.

## How to add a new chapter

1. Add a record to `sample-wrapped-2026.json` with at least `chapter_slug`, `chapter_name`, `region_name`, and `year_label`.
2. Optionally add a config entry in `wrapped-config-2026.json` under `chapters[slug]`.
3. Run `node check-production.js` — it will generate the share page for the new chapter.
4. Commit all changed files including the new `share/{slug}/index.html`.

## How to add a national/region/program story

Use `scope_type: "national"` / `"region"` / `"program"`, `scope_slug`, and `scope_name` fields instead of `chapter_slug`/`chapter_name`. See `docs/data-contract.md` for the full field list.

## Social image configuration

The default social preview image is the hardcoded `SOCIAL_IMAGE_URL` constant in `jsu-wrapped.js`. To override it per-embed (e.g., for WordPress vs. GitHub Pages), set `data-social-image` on the `#jsu-wrapped` container:
```html
<div id="jsu-wrapped" data-social-image="https://ncsy.org/wp-content/uploads/wrapped-preview.png" ...>
```
Per-chapter social images can be pre-generated at build time using the SVG fallback logic in `createFallbackSvg` — see `docs/superpowers/plans/` for the planned implementation.

## URL params handled by the widget

| Param | Effect |
|---|---|
| `?chapter=baltimore` | Load chapter story |
| `?scope=national` | Load national story |
| `?scope=region&region=atlantic-seaboard` | Load region story |
| `?mode=teen&teen=...` | Load teen story (noindex) |
| `?variant=donor-recap` | Apply variant overlay |
| `?card=3` | Start on a specific card index |
| `?autoplay=true` | Enable autoplay |

## Known gotchas

- **`jsu-wrapped.js` is ES5** — no `const`, `let`, arrow functions, template literals, or destructuring. `wrapped-builder.js` and all Node tooling use modern JS. Do not mix.
- **`cleanPublicText` line ~184** has a hardcoded `"Jewniors" → "Juniors"` correction for a known CRM data-quality issue.
- **`replaceChangedMetricValues`** substitutes the original metric number in card copy when `record_overrides` corrects it. It sorts replacements longest-first to reduce false positives but can still collide for numbers sharing digit patterns. Review carefully when adding record overrides with numbers that appear elsewhere in copy.
- **Google Fonts dependency**: `jsu-wrapped.css` references Bricolage Grotesque, DM Sans, and Caveat but does not load them. The host page must include the `<link>` to Google Fonts.
- **Teen mode is noindex proof-of-concept** — do not use real teen-level identifying data until privacy review is complete. The validator blocks ID/contact fields.
- **Builder `data-review-email`** must be populated before distributing to pilot staff. Use the Reviewer setup panel and "Copy staff link" to generate pre-addressed builder links.
- **`fetchConfig` logs a warning** (not an error) if the config URL fails. Check the browser console if config overrides aren't applying.

## Validator key sets (quick reference)

Allowed top-level config section keys are defined in `CONFIG_SECTION_KEYS` in `validate-wrapped-data.js`. Any key not in that set causes a validation error. Currently includes: `label`, `name`, `title`, `description`, `hidden_from_picker`, `hiddenFromPicker`, `brand_logo`, `palette`, `accent_palette`, `cta_label`, `ctaLabel`, `cta_target`, `ctaTarget`, `cta_href`, `ctaHref`, `hidden_cards`, `card_overrides`, `record_overrides`, `custom_cards`, `variants`, `footprint_label`.

Card override keys: `eyebrow`, `displayEyebrow`, `headline`, `displayHeadline`, `subtext`, `badge`, `markerText`, `persona`, `chapterName`, `schoolName`, `yearLabel`.
